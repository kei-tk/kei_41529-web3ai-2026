# 第7回 — プロトタイプ v2「棚(Tana)」🗄️

## 一言ピッチ
**散らかった Downloads を、AIが中身を読んで自動で“棚”に戻す整理ダッシュボード。**

## プロトタイプ v2 のデプロイURL
👉 https://kei-tk.github.io/kei_41529-web3ai-2026/

## v1 からの差分(一言)
**「自分で手動フォルダ分け+ファイル名検索」だった v1 を、「AIが中身を読んで自動で棚分け+中身まで横断検索」できる v2 にした。**

## 自分の感想・気付き
自分で触ってみて、機能の正しさより **「散らかった山が一瞬で棚に収まる瞬間の気持ちよさ」** が一番効くと気付いた。
v1 で一番不満だった「**名前でしか探せない**」も、中身を読んで横断検索できるようにしただけで本当に解消され、
検索が「思い出す作業」から「**打てば出る**」に変わった手応えがあった。

まだ、LLM などを使っているわけではないので、ファイルの内容などを正確に分析して仕分けできるわけではないのが、少し不便だと感じた。
LLM などを使って、仕分けることのできる幅を広げれば、もっと便利になると感じた。

---

## 補足

### これは何か
第3回の VPC で選んだバグ **「ファイル整理」** をそのまま形にしたプロダクト。
VPC の「DLした瞬間にAIが中身を読んで自動分類」+「日付・科目・キーワードで絞れるダッシュボード」を実装した。

### v1 → v2 の差分(詳細)
| | v1(素朴版) | v2「棚」 |
|---|---|---|
| 仕分け | 自分で手動 | **AIが中身を読んで自動で棚へ** |
| 検索 | ファイル名のみ | **名前+中身+タグを横断**全文検索 |
| 思い出すヒント | ファイル名だけ | **AIの一言サマリ + 中身プレビュー + 確信度** |
| 絞り込み | なし | **棚・科目・期間・タグ**フィルタ |
| 取り込み | 1つずつ | **フォルダごと / ドラッグ&ドロップ**で一括 |

### AI について(正直な説明)
アプリ内の分類エンジン(`tana/src/lib/classify.ts`)は、本物の LLM ではなく
**キーワード辞書ベース**で「中身を読んで意味で仕分ける」体験を再現したもの。
`classify()` を 1 関数差し替えるだけで Gemini 等の API に置換できる設計。
※「AIと一緒に作る」課題の趣旨どおり、開発自体は Claude / Antigravity を活用して進めた。
